¿Qué es la Alta Disponibilidad (HA)?

La Alta Disponibilidad (HA) es una característica de los sistemas informáticos que garantiza un tiempo de actividad elevado y minimiza las interrupciones del servicio. HA es fundamental en sistemas críticos que necesitan estar siempre operativos.

¿Por qué es importante?

En sectores como el financiero, salud o telecomunicaciones, el tiempo de inactividad puede tener consecuencias graves. HA asegura que los servicios están disponibles casi de forma continua, incluso en caso de fallos de hardware, red o software.

Principios de HA

  • Redundancia: Se duplican componentes para que, si uno falla, el otro tome el control.
  • Balanceo de carga: Distribuye las tareas entre múltiples servidores para evitar sobrecargas.
  • Conmutación por error: Cuando un servidor falla, otro asume sus funciones automáticamente.

Ejemplo de Configuración HA

En una configuración simple, tenemos un sistema master-slave, donde el nodo master maneja el tráfico y el nodo slave toma el control si el master falla. Este proceso de transferencia de tareas entre nodos es la conmutación por error.

Ambos nodos comparten una IP virtual, que siempre apunta al nodo activo. Cuando el nodo master se detiene, el slave asume la IP virtual, asegurando que el servicio sigue disponible.
